Looks like the lighttpd service is working.

Configuration needed

Systemd portable service

In order to serve something useful with the portable lighttpd service you must bind the document root directory to /srv/www. This done by adding a drop-in with a BindReadOnlyPaths=, e.g.: 
# systemctl edit lighttpd.service
[Service]
BindReadOnlyPaths=<your-html-dir>:/srv/www
Then restart the service and you should see your documents.

Docker container

When running the docker container you should add the proper -v option to the command line, e.g.: 
$ docker run -v <your-html-dir>:/srv/www lighttpd ...
